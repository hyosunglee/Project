# 🎯 사용 방법

## ✅ 해결됨: Console에서 기능 동작 문제

이제 **3가지 방법**으로 시스템을 사용할 수 있습니다:

---

## 📱 방법 1: 웹 UI 사용 (가장 쉬움!)

1. **Replit의 Webview 탭을 엽니다**
2. 웹 페이지가 자동으로 열립니다
3. 버튼을 클릭하여 기능을 사용합니다:
   - **서버 상태 확인** 버튼으로 시스템 확인
   - **테스트 데이터 생성** 버튼으로 샘플 데이터 생성
   - **모델 학습 시작** 버튼으로 AI 모델 학습
   - **예측 실행** 버튼으로 텍스트 분석
   - **데이터 추가** 버튼으로 새 데이터 입력

### 장점:
- ✅ 가장 쉽고 직관적
- ✅ 버튼 클릭만으로 모든 기능 사용
- ✅ 실시간으로 결과 확인

---

## 💻 방법 3: curl 명령어 사용 (고급)

Shell 탭에서 직접 API를 호출할 수 있습니다:

### 서버 상태 확인
```bash
curl http://localhost:5000/healthz
```

### 테스트 데이터 생성
```bash
curl -X POST "http://localhost:5000/seed?n=20"
```

### 모델 학습
```bash
curl -X POST http://localhost:5000/train
```

### 예측 실행
```bash
curl -X POST http://localhost:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"text":"새로운 강화학습 알고리즘", "target":"reward", "explain":true}'
```

### 데이터 추가
```bash
curl -X POST http://localhost:5000/ingest \
  -H "Content-Type: application/json" \
  -d '{"title":"논문 제목", "text":"논문 내용", "label": 1}'
```

## 🔄 워크플로우 예시

### 기본 사용 흐름:

1. **웹 UI 열기** → Webview 탭
2. **서버 상태 확인** → "서버 상태 확인" 버튼
3. **테스트 데이터 생성** → 개수 입력 후 "테스트 데이터 생성" 버튼
4. **모델 학습** → "모델 학습 시작" 버튼 (백그라운드에서 실행)
5. **예측 테스트** → 텍스트 입력 후 "예측 실행" 버튼

### 실제 논문 수집:

1. **Shell 탭**에서 실행:
```bash
python collector.py
```

2. **웹 UI**에서 "모델 학습 시작" 버튼 클릭

3. 학습 완료 후 "예측 실행"으로 테스트

## 🆘 문제 해결

### 웹 UI가 안 보이는 경우:
- Replit 상단의 **Webview** 탭을 클릭하세요

### 서버가 안 돌아가는 경우:
1. Shell에서 확인: `curl http://localhost:5000/healthz`
2. 재시작: 워크플로우를 다시 시작하세요

### 모델 학습이 안 되는 경우:
1. 먼저 데이터를 생성하세요 (최소 10개 이상)
2. 웹 UI에서 "테스트 데이터 생성" 버튼 클릭
3. 그 다음 "모델 학습 시작" 버튼 클릭
